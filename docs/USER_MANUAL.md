## 1. เข้าสู่หน้าโครงการของตัวเอง

1. เปิดเว็บไซต์ของระบบ
2. ไปที่ **โครงการย่อย** (`/projects`)
3. ค้นหาด้วยชื่อโครงการ หรือชื่อผู้รับผิดชอบ
4. คลิกเข้าไปในโครงการของตัวเอง → `/projects/[id]`

ในหน้านี้จะเห็น:
- 📋 ข้อมูลทั่วไป (งบ, ผู้รับผิดชอบ, พื้นที่)
- 💰 **BudgetReconciliation** — งบ 3 แหล่ง (ได้รับ / เบิก ERP / รายงานสะสม)
- 📊 **S-Curve** กราฟแผน vs จริง
- ✅ กิจกรรม · ตัวชี้วัด · รายงานที่ส่งไปแล้ว

## 2. ใส่ Token เพื่อรายงาน

ในหน้ารายละเอียดโครงการ จะมีช่องกรอก **Token** ด้านบน

- ใส่รหัส 6 หลัก (ตัวอย่าง: `A3X9K2`) → กดยืนยัน
- เมื่อ Token ถูกต้อง → ปุ่ม **รายงานผล** และ **✏️ แก้ไข** จะปรากฏ
- Token จะถูกจำไว้ในเบราว์เซอร์ → ไม่ต้องกรอกใหม่ทุกครั้ง

> ⚠️ **Token เป็นของส่วนตัว — อย่าแชร์ให้คนอื่น** เพราะใครมี Token ก็แก้ไขข้อมูลโครงการของคุณได้

## 3. ส่งรายงานกิจกรรมครั้งแรก

### ขั้นตอน

1. คลิกปุ่ม **📝 รายงานผล** ในกิจกรรมที่ต้องการรายงาน
2. กรอกฟอร์ม:

   | ช่อง | ต้องกรอก? | คำแนะนำ |
   |---|---|---|
   | คำอธิบายผลดำเนินงาน | ✅ บังคับ | สรุปผลกิจกรรม 3-5 บรรทัด |
   | ยอดเงินที่ใช้ (budget_spent) | 💡 ควรกรอก | กรอกเฉพาะยอดของกิจกรรมนี้ (หน่วย: บาท) |
   | ลิงก์หลักฐานหลัก | 💡 ควรกรอก | Drive/Dropbox ของเอกสารรายงาน |
   | หลักฐานแนบ (หลายไฟล์) | เสริม | เพิ่มได้หลายรายการ · เลือกประเภท (image/pdf/link/video) |
   | ตอบตัวชี้วัด (KPI) | เสริม | ใส่ค่าจริง → ถ้าถึงเป้าจะขึ้น ✅ Verified |
   | SDG tags | เสริม | เลือก 1-17 (SDG ที่เกี่ยวข้อง) |
   | เพิ่มตัวชี้วัดใหม่ | เสริม | ถ้ามี KPI นอกเหนือจากที่วางแผนไว้ |
   | ผู้ส่งรายงาน | ✅ บังคับ | ชื่อ-นามสกุลของคุณ |

3. กด **ส่งรายงาน**
4. ระบบจะ:
   - บันทึก `activity_reports`
   - อัปเดตสถานะกิจกรรม → `completed` / `in_progress`
   - รวม SDG tags เข้าโครงการ
   - คำนวณ RPF coin (ถ้ารายงานตรงเวลา) → เพิ่ม reward_balance

### ตัวอย่างการกรอกหลักฐานแนบ

กดปุ่ม **+ เพิ่มหลักฐาน** → เลือกประเภท → วาง URL:
```
[Image]  https://drive.google.com/.../photo.jpg
[PDF]    https://drive.google.com/.../rayngan.pdf
[Link]   https://facebook.com/.../post
[Video]  https://youtube.com/watch?v=...
```

## 4. แก้ไขรายงานที่ส่งไปแล้ว

ถ้าส่งรายงานไปแล้วแต่ต้องแก้ **ไม่ต้องแจ้ง Admin** — หน.โครงการแก้เองได้

### 4.1 แก้ยอดเงินเร็ว (Inline)

ในกล่องรายงานจะมี **💰 แก้ยอด** — กดแล้วพิมพ์ยอดใหม่ → **บันทึก**

### 4.2 แก้ทุกอย่าง (ผ่าน Modal)

1. กดปุ่ม **✏️ แก้ไข** บนรายงานที่ต้องการ
2. Modal เปิดขึ้นมา — แก้ได้ทั้ง:
   - คำอธิบาย
   - ยอดเงินที่ใช้
   - ลิงก์หลักฐานหลัก
   - หลักฐานแนบ (เพิ่ม/ลบ/เปลี่ยนประเภทได้)
3. กด **บันทึกการแก้ไข**
4. ระบบจะ:
   - อัปเดต `activity_reports`
   - ถ้ายอดเงินเปลี่ยน → คำนวณ `budget_reported` ใหม่อัตโนมัติ
   - ถ้าใส่ SDG tags ใหม่ → merge เข้าโครงการ

### ข้อจำกัด

- Token ต้องตรงกับโครงการของรายงานนั้น (ป้องกันแก้ของคนอื่น)
- ข้อมูล KPI contributions และ participants ของรายงานเดิม **ยังคงอยู่** — ไม่ถูกแก้จาก modal นี้

## 5. เข้าใจตัวเลขงบประมาณ 3 ตัว

ในหน้าโครงการจะเห็นกล่อง **BudgetReconciliation** แสดงงบ 3 ตัว:

```
┌─────────────────────────────────────────┐
│  งบที่ได้รับ (total)       400,000 บาท  │  ← จัดสรรจากมหาวิทยาลัย
│  เบิกจาก ERP (used)        211,850 บาท  │  ← ยอดที่หักจาก ERP แล้ว
│  รายงานสะสม (reported)     200,000 บาท  │  ← SUM(ยอดในรายงาน)
│  งบคงเหลือ (remaining)     188,150 บาท  │  = total − max(used, reported)
└─────────────────────────────────────────┘
```

### ความต่างของ used vs reported

| สถานการณ์ | used | reported | ความหมาย |
|---|---|---|---|
| ปกติ | ≤ | = 0 | ยังไม่มีการรายงาน — แค่เบิกแล้ว |
| หน.รายงาน | ≤ | > 0 | รายงานแล้ว แต่ยังไม่มากกว่า ERP |
| หน.ออกเงินเอง | < | > | `advance = reported − used` → รอ ERP ตามจ่าย |

> 💡 **ถ้าเห็น budget_remaining ติดลบ หรือรู้สึกผิดปกติ** → แจ้ง Admin ให้รัน **Repair budget** tool

## 6. FAQ คำถามที่พบบ่อย

<details>
<summary><b>Token หาย ทำยังไง?</b></summary>

ติดต่อ Admin — Admin สามารถดู token เดิมได้จาก `/admin → Tokens`
หรือ Generate token ใหม่ได้ (token เก่าจะถูก deactivate)

</details>

<details>
<summary><b>รายงานแล้วยอดงบในหน้าโครงการไม่เปลี่ยน</b></summary>

- ลอง refresh หน้า (บางครั้ง cache ค้าง)
- ตรวจช่อง **ยอดเงินที่ใช้** ในรายงาน — ถ้าเป็น 0 → `budget_reported` จะไม่เพิ่ม
- ใช้ **💰 แก้ยอด** inline edit ใส่ค่าใหม่

</details>

<details>
<summary><b>อัปโหลดรูปภาพ/ไฟล์ได้ไหม?</b></summary>

ระบบ**ไม่มี file storage ของตัวเอง** — ใช้วิธีวาง URL แทน

แนะนำ:
1. อัปโหลดไฟล์ขึ้น **Google Drive** / **Dropbox** / **OneDrive** — ตั้ง share = "ใครมีลิงก์ดูได้"
2. Copy URL มาวางในช่องหลักฐาน
3. เลือกประเภท image / pdf / link / video ให้ตรง

</details>

<details>
<summary><b>เพิ่มตัวชี้วัดใหม่ที่ไม่ได้เขียนไว้ใน ง.9 ได้ไหม?</b></summary>

ได้ — ในฟอร์มรายงานมีส่วน **"เพิ่มตัวชี้วัดใหม่"**
KPI ใหม่จะถูก flag `is_additional = true` เพื่อให้ Admin/ผู้ตรวจทราบว่าเป็น KPI ที่เพิ่มระหว่างโครงการ

</details>

<details>
<summary><b>ส่งรายงานซ้ำซ้อน/ผิด จะลบได้ไหม?</b></summary>

**ลบเองไม่ได้** — ติดต่อ Admin เพื่อลบผ่าน SQL Editor ใน Supabase Dashboard
หรือแก้ด้วย ✏️ Modal ให้ถูกต้อง (แนะนำ — เก็บประวัติดีกว่า)

</details>

<details>
<summary><b>RPF coin ได้เมื่อไร?</b></summary>

ได้อัตโนมัติเมื่อส่งรายงาน (มี Token) — ตามกฎ:

| ประเภท | เงื่อนไข | RPF |
|---|---|---|
| on_time | ส่งในเดือนที่วางแผน | +10 |
| early_bonus | ส่งก่อนครบเดือน | +5 |
| consistency_bonus | ส่งติดต่อกัน 3 ครั้ง | +15 |
| kpi_verified | ค่าจริง ≥ เป้า | +20 |
| kpi_exceeded | ค่าจริง > เป้า × 1.5 | +30 |

ดูยอดสะสมที่ `/admin → Rewards` (Admin ดูให้ได้)

</details>
